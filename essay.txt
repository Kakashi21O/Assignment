5th June In-Lecture Practice Questions

Q.1 What is the role of `.git` folder?

The `.git` folder is the main folder used by Git to store all version control information.

It stores:

* Commit history
* Branches
* Logs
* Configuration settings
* File tracking data

Without the `.git` folder, Git cannot track changes in the project.

Q.2 What is Staging Environment? Use of `git add`

The staging environment is a temporary area where files are prepared before making a commit.

Command:

git add filename

Example:

git add app.py

To add all files:

git add .

Purpose:

`git add` moves selected changes into the staging area for the next commit.

Q.3 Usage of `git remote add origin <url>`

This command connects the local repository to a remote repository like GitHub.

Example:

git remote add origin https://github.com/user/project.git

Here:

* `origin` → name of remote repository
* `<url>` → repository link

Purpose:

Allows pushing and pulling code from GitHub.

Q.4 Usage of `git remote -v`

This command shows connected remote repository URLs.

Command:

git remote -v

Example Output:

origin  https://github.com/user/project.git (fetch)
origin  https://github.com/user/project.git (push)

Purpose:

Used to check connected remote repositories.

Q.5 Difference between `git add` and `git commit`

`git add`

* Moves files to staging area

`git commit`

* Saves staged changes permanently in Git history

Example:

git add .
git commit -m "Added login feature"

Simple Meaning:

* `git add` → Prepare changes
* `git commit` → Save changes

Q.6 Use of `git push --upstream`

Command:

git push --upstream origin main

Purpose:

* Pushes code to GitHub
* Connects local branch with remote branch

After using it once, future pushes can be done using:

git push

Q.7 Difference between `git push --upstream origin main` and `git push origin main`

`git push --upstream origin main`

* Pushes code
* Sets tracking connection

`git push origin main`

* Only pushes code

`--upstream` is usually used the first time.

Q.8 Importance of `README.md` file

`README.md` is the documentation file of a project.

It contains:

* Project name
* Features
* Installation steps
* Usage instructions
* Author information

Importance:

It helps users understand the project easily.

Q.9 Meaning of `U`, `A`, `M`, `D` in Git Status

Symbols in `git status`:

* `U` → Untracked file
* `A` → Added file
* `M` → Modified file
* `D` → Deleted file

Command:

git status

Q.10 Linux command to automatically log date and time during first startup

Open crontab:

crontab -e

Add this line:

@reboot date >> startup_log.txt

Meaning:

* `@reboot` → runs when system starts
* `date` → shows current date and time
* `>>` → appends output to file

Example Output:

Sun Jun 7 11:30:00 IST 2026
